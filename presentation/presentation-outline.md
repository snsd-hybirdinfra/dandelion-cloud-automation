## 2. 발표 1번 - 정주헌

### 슬라이드 제목

프로젝트 개요 및 전체 아키텍처

### 발표 핵심

- 프로젝트 주제 소개
- 프로젝트 목표 설명
- 팀원별 역할 소개
- 전체 시스템 아키텍처 설명
- Ansible Control Node 중심 구조 설명

### 발표 멘트

안녕하십니까. PM 및 아키텍처를 담당한 정주헌입니다.

이번 프로젝트의 주제는 Ansible 기반 클라우드 인프라 자동화 및 운영 최적화 시스템 구축입니다.

본 프로젝트는 클라우드 서버의 초기 설정, Docker 기반 서비스 배포, 상태 점검, 백업 및 복구 검증 과정을 Ansible을 통해 자동화하는 것을 목표로 합니다.

전체 구조는 Ansible Control Node를 중심으로 설계했습니다.

Control Node에서 Inventory와 Playbook을 관리하고, SSH Key 기반으로 Managed Node에 접속하여 서버 설정과 서비스 배포를 수행합니다.

팀원별로는 클라우드 인프라, 서버 및 Docker 구성, Ansible 자동화, 모니터링과 백업 검증으로 역할을 나누었고, 저는 이 과정들이 하나의 운영 자동화 흐름으로 연결되도록 전체 구조와 발표자료를 통합했습니다.

### 화면에 보여줄 자료

- README.md 프로젝트 개요
- docs/architecture.md 아키텍처 다이어그램
- 팀원 역할표

## 3. 발표 2번 - 백서빈

### 슬라이드 제목

클라우드 인프라 구성

### 발표 핵심

- 클라우드 플랫폼
- 서버 구성
- IP 정보
- 네트워크 구성
- 보안그룹 정책

Control Node, Web Node, Backup Node가 어떤 IP와 역할을 갖는지 설명하고, SSH와 HTTP 접근을 위한 보안그룹 정책을 보여준다.

### 화면에 보여줄 자료

- 클라우드 인스턴스 목록 캡처
- 네트워크 / 서브넷 캡처
- 보안그룹 캡처
- SSH 접속 성공 캡처

서버 및 Docker 구성

Docker 설치 후 Nginx 컨테이너를 실행하고, curl 명령어로 웹 서비스가 정상 동작하는지 확인한다.

## 5. 발표 4번 - 조민석

### 슬라이드 제목

Ansible 자동화 구현

### 발표 핵심

- Ansible Control Node 구성
- SSH Key 기반 접속
- inventory.ini 작성
- ansible.cfg 작성
- site.yml Playbook 작성
- ansible ping 테스트
- ansible-playbook 실행 결과

### 발표 가이드

Ansible 자동화 파트에서는 수동으로 진행하던 서버 설정 작업을 Playbook으로 자동화한 과정을 설명한다.

Inventory 파일에는 관리 대상 서버 정보를 정의하고, site.yml Playbook을 통해 기본 패키지 설치, Docker 설치, Docker 서비스 실행, Nginx 컨테이너 배포를 자동화한다.

### 화면에 보여줄 자료

- ansible --version 캡처
- inventory.ini 캡처
- ansible.cfg 캡처
- site.yml 캡처
- ansible all -m ping 성공 캡처
- ansible-playbook 실행 성공 캡처

## 6. 발표 5번 - 박재우

### 슬라이드 제목

모니터링 / 백업 / 복구 검증

### 발표 핵심

- 서버 상태 점검
- Docker 상태 확인
- Nginx 서비스 확인
- 백업 스크립트 실행
- 복구 테스트
- 검증 체크리스트

### 발표 가이드

검증 파트에서는 Ansible 자동화 이후 서버와 서비스가 정상적으로 동작하는지 확인한다.

health_check.sh를 통해 서버 상태, Docker 상태, 컨테이너 상태, HTTP 응답을 확인하고, backup.sh를 통해 웹 데이터 백업을 생성한다.

이후 테스트 파일 삭제 후 백업 파일을 이용해 복구가 가능한지 검증한다.

### 화면에 보여줄 자료

- health_check.sh 실행 결과
- Docker 상태 캡처
- HTTP 확인 캡처
- 백업 파일 생성 캡처
- 복구 성공 캡처
- Validation Checklist

### 슬라이드 제목

최종 결과 및 기대효과

### 발표 핵심

- 전체 구현 흐름 요약
- Ansible 자동화 결과 정리
- 검증 결과 정리
- 프로젝트 기대효과
- 마무리

### 발표 멘트

이번 프로젝트는 단순히 서버를 구축하는 것이 아니라, 반복적인 수동 서버 설정 작업을 Ansible Playbook으로 자동화하고, 그 결과를 상태 점검, 백업, 복구 검증까지 연결했다는 점에서 의미가 있습니다.

클라우드 인프라 준비부터 서버 환경 구성, Ansible 자동화, Docker 서비스 배포, 상태 점검, 백업 및 복구 검증까지 하나의 운영 흐름으로 구성했습니다.

이를 통해 인프라 운영에서 반복되는 작업을 자동화하고, 자동화 결과를 검증 가능한 형태로 남기는 기본적인 운영 자동화 구조를 구현했습니다.

이상으로 발표를 마치겠습니다. 감사합니다.

## 8. 최종 발표 메시지

~~~text
인프라 준비
→ 서버 환경 구성
→ Ansible 자동화
→ Docker 서비스 배포

→ 백업 / 복구 검증
~~~
