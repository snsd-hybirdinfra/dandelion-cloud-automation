## 3. 담당자별 진행상황

### 3.1 조민석 - Ansible Automation / Restore

| 항목 | 상태 |
|---|---|
| Restore 자동화 구현 | 완료 |
| Restore 자동화 검증 | 예정 |

#### 상세 내용

Restic 기반 복구 흐름과 연계하여 Restore 자동화 구현을 완료하였다.

현재는 구현된 Restore 자동화가 실제 복구 상황에서 정상적으로 동작하는지 검증할 예정이다.

#### 발표 설명 기준

~~~text
Restore 자동화는 구현을 완료했습니다.
이후 실제 장애 또는 설정파일 손상 상황에서 Restore 자동화가 정상 동작하는지 검증할 예정입니다.
~~~

### 3.2 이진욱 - DB / Service

| 항목 | 상태 |
|---|---|
| DB 이중화 | 완료 |
| DB Failover 테스트 | 완료 |
| DB 복구 점검 | 완료 |

#### 상세 내용

MariaDB Primary - Replica 기반 DB 이중화 구성 이후 Failover 테스트와 복구 점검을 완료하였다.

이를 통해 DB 장애 발생 시 전환 및 복구 흐름을 검증하였다.

#### 발표 설명 기준

~~~text
DB는 Primary - Replica 기반 이중화 구성을 완료했고,
Failover 테스트와 복구 점검까지 완료했습니다.

최종 발표에서는 DB 가용성 검증 완료 항목으로 설명할 수 있습니다.
~~~
