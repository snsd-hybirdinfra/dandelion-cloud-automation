## 3. 주요 진행 내용

### 3.1 이진욱 - Phase 1 완료 및 Phase 2 작업 진행

이진욱은 Phase 1 작업을 완료하고, Phase 2 작업을 진행 중한 뒤 작업 완료 후 관련 문서 작업을 진행할 예정이다.

| 작업 항목 | 진행 내용 | 상태 |
|---|---|---|
| Phase 1 작업 | 기본 서비스 구성 및 초기 구축 작업 완료 | 완료 |
| Phase 2 작업 | 운영 확장 범위 작업 진행 완료 | 완료 |
| 문서 작업 | Phase 1 / Phase 2 작업 완료 후 문서화 진행 | 진행 |

Phase 1 완료를 통해 기본 인프라 및 서비스 구성 범위가 마무리되었으며, Phase 2 작업 진행 중 후 결과 정리를 위한 문서 작업을 진행하고 있다.

### 3.2 조민석 - Phase 1 파일 확인 및 Ansible Playbook 보완

조민석은 이진욱이 진행한 Phase 1 파일을 확인한 뒤, 기본 설정을 수정하고 관련 Playbook 작성을 완료하였다.

또한 박재우가 작성한 Phase 1 파일을 확인한 뒤, 백업 Playbook을 작성하고 현재 포트 충돌 문제를 해결 중이다.

| 작업 항목 | 진행 내용 | 상태 |
|---|---|---|
| 이진욱 Phase 1 파일 확인 | 서버 / DB / Proxy 관련 Phase 1 파일 확인 | 완료 |
| 기본 설정 수정 | Ansible 실행 기준에 맞춰 기본 설정 수정 | 완료 |
| Playbook 작성 | Phase 1 구성 반영 Playbook 작성 완료 | 완료 |
| 박재우 Phase 1 파일 확인 | Backup / Monitoring / Script 관련 파일 확인 | 완료 |
| Backup Playbook 작성 | 백업 자동화 Playbook 작성 | 완료 |
| 포트 충돌 대응 | 백업 또는 검증 과정에서 발생한 포트 충돌 해결 중 | 진행 |

현재 조민석 작업은 단순 코드 작성이 아니라, 팀원별 Phase 1 결과물을 확인한 뒤 Ansible 자동화 기준에 맞게 정리하고 보완하는 역할이다.

포트 충돌 문제는 실제 실행 또는 검증 과정에서 발생할 수 있는 운영 이슈이므로, 해결 후 트러블슈팅 항목으로 정리할 예정이다.

## 7. 회의 결과 요약

2026년 6월 25일 회의에서는 Phase 1 완료 상태와 Phase 2 진행 상태를 확인하였다.

이진욱은 Phase 1 작업을 완료하고 Phase 2 작업을 진행 중하였으며, 작업 완료 후 관련 문서 작업을 진행할 예정이다.

조민석은 이진욱의 Phase 1 파일을 확인한 뒤 기본 설정을 수정하고 Playbook 작성을 완료하였다. 또한 박재우의 Phase 1 파일을 확인한 뒤 Backup Playbook을 작성하였으며, 현재 포트 충돌 문제를 해결 중이다.

박재우는 Phase 1 작업을 완료하고, 현재 제출 및 검증 자료 확보를 위한 캡처 작업을 진행 중이다.

금일 기준 Phase 1은 주요 담당 영역에서 완료 상태로 정리되었으며, 후속 작업은 Phase 2 문서화, Backup Playbook 포트 충돌 해결, 제출용 캡처 정리, Playbook 실행 검증 중심으로 진행할 예정이다.
