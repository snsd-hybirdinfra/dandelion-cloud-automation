## 4. 구성 변경 작업

### 4.1 Web Node Docker Swarm Cluster 구성 변경

기존 Web Node 구성은 단일 Web Node 또는 Web1 / Web2 개별 구성 중심이었으나, 29일 기준으로 Docker Swarm Cluster 기반 구성으로 수정한다.

| 항목 | 기존 구성 | 변경 구성 |
|---|---|---|
| Web 서비스 | 단일 Web 또는 개별 Web Node | Docker Swarm Cluster |
| 배포 방식 | 개별 컨테이너 실행 | Swarm Service 기반 배포 |
| 확장 방식 | Web Node 추가 | Cluster 기반 확장 |
| 장애 대응 | 개별 노드 장애 대응 | Cluster 서비스 기반 대응 |

Docker Swarm Cluster를 통해 Web 서비스 영역의 확장성과 운영 안정성을 높이는 방향으로 변경한다.

---

### 4.2 Proxy Node 역할 변경

Proxy Node는 Load Balancer 역할을 제거하고 Reverse Proxy 역할만 수행하도록 변경한다.

| 항목 | 기존 구성 | 변경 구성 |
|---|---|---|
| Proxy Node 역할 | Reverse Proxy + Load Balancer | Reverse Proxy 전담 |
| Load Balancer 역할 | Proxy Node에서 수행 | Web Node / Web Cluster 영역에서 수행 |
| 요청 흐름 | Proxy → Web1 / Web2 | Proxy → Web Cluster |

변경 후 요청 흐름은 다음과 같다.

~~~text
User
→ Proxy Node Reverse Proxy
→ Web Cluster / Load Balancing 영역
→ WordPress Service
~~~
