# 5. mysqld_exporter 계정 복제 문제 해결

Monitoring 자동화 중 DB Primary와 DB Replica 양쪽에서 `mysqld_exporter@localhost` 계정을 직접 생성하면서 Replication SQL Thread가 중단되는 문제가 발생하였다.

오류 내용:

```text
Last_SQL_Errno: 1396
Last_SQL_Error: Operation CREATE USER failed for 'mysqld_exporter'@'localhost'
Slave_IO_Running: Yes
Slave_SQL_Running: No
```

원인:

```text
Primary에서 CREATE USER 실행
→ 해당 SQL이 Replica로 복제됨
→ 그런데 Replica에도 이미 같은 계정이 직접 생성되어 있음
→ CREATE USER 충돌
→ Replica SQL Thread 중단
```

해결 방향:

```text
mysqld_exporter 계정은 Primary에서만 생성
Replica는 Replication을 통해 계정이 복제될 때까지 대기
```

Replica SQL Thread는 Skip Counter로 복구하였다.

```bash
ansible db-replica-node -b -m shell -a "
mariadb -e \"
STOP SLAVE SQL_THREAD;
SET GLOBAL SQL_SLAVE_SKIP_COUNTER=1;
START SLAVE SQL_THREAD;
\"
"
```

이후 Replication 정상 상태를 확인하였다.

```text
Slave_IO_Running: Yes
Slave_SQL_Running: Yes
Seconds_Behind_Master: 0
Last_IO_Error: 없음
Last_SQL_Error: 없음
```

# 6. Check Mode 대응 수정

`site.yml --check --diff` 실행 중 여러 조회/검증 Task가 Check Mode에서 실제 명령을 실행하지 않아 실패하였다.

대표 증상:

```text
msg: Command would have run if not in check mode
stdout: ""
until 조건 실패
```

수정 원칙:

```text
상태 조회 / 검증 / 대기 Task
→ check_mode: false
→ changed_when: false

실제 변경 / 배포 / 재시작 Task
→ Check Mode 기본 동작 유지
또는 when: not ansible_check_mode 사용
```

수정 대상 예시:

```text
MariaDB 포트 응답 대기
MariaDB 버전 확인
Primary WordPress DB 확인
Replica Replication 상태 확인
mysqld_exporter 계정 복제 대기
WordPress Replica 실행 상태 대기
WordPress HTTP 응답 확인
HAProxy HTTPS 응답 확인
Monitoring 서비스 상태 확인
```

핵심은 Check Mode에서도 실제 인프라 상태를 읽어야 하는 검증 Task는 실행되도록 한 것이다.

# 11. HAProxy DB TCP Proxy 누락 해결

`templates/haproxy.cfg.j2`를 확인한 결과 WordPress용 80/443 설정만 있고 MariaDB용 3306 TCP Proxy 설정이 빠져 있었다.

기존 HAProxy Template 구성:

```text
frontend wordpress_frontend
backend wordpress_swarm_backend
```

누락된 구성:

```text
frontend mariadb_frontend
backend mariadb_backend
```

추가한 설정:

```haproxy
frontend mariadb_frontend
    bind *:{{ mariadb_port }}
    mode tcp
    option tcplog
    default_backend mariadb_backend


backend mariadb_backend
    mode tcp
    option tcp-check
    server db-primary {{ hostvars[groups['db'][0]].ansible_host }}:{{ mariadb_port }} check inter 3s fall 3 rise 2
    server db-replica {{ hostvars[groups['db_replica'][0]].ansible_host }}:{{ mariadb_port }} check inter 3s fall 3 rise 2 backup
```

해당 설정의 역할:

```text
Proxy Node 192.168.4.32:3306
→ DB Primary 192.168.4.195:3306
→ Primary 장애 시 DB Replica 192.168.4.222:3306 backup
```

이 설정을 Template에 영구 반영하여 이후 `site.yml` 실행 시에도 DB Proxy 설정이 유지되도록 하였다.
