# Phase 1 Ansible 실행 및 검증 정리
## 공통 환경 적용부터 WordPress 80번 포트 충돌 발생까지

## 1. 문서 목적

본 문서는 기존 `26_06_24.md` 이후 진행한 Ansible Playbook 실행 및 검증 내용을 정리한다.

이번 작업 범위는 다음과 같다.

```text
common.yml 실행 및 검증
→ docker.yml 실행 및 검증
```

# 5. database.yml 실행 준비 중 오류 발생

## 5.1 문법 검사 오류

```bash

```

다음 오류가 발생하였다.

```text
ERROR! couldn't resolve module/action 'ansible.mysql.mysql_user'
```

오류 위치:

```yaml
- name: MariaDB 익명 사용자 제거
  ansible.mysql.mysql_user:
```

## 5.2 원인

`database.yml`에서 사용하는 다음 모듈은 `ansible.mysql` Collection에 포함되어 있다.

```text
ansible.mysql.mysql_user
ansible.mysql.mysql_db
```

Control 노드에 해당 Collection이 설치되지 않아 모듈을 찾지 못한 것이 원인이었다.

# 6. requirements.yml 작성 및 Collection 설치

## 6.1 requirements.yml 작성

파일 위치:

```text
~/dandeliondir/ansible/requirements.yml
```

작성 내용:

```yaml
---
collections:
  - name: ansible.mysql
  - name: community.docker
```

Collection 용도:

| Collection | 사용 위치 |
|---|---|
| `ansible.mysql` | `database.yml`의 MariaDB DB 및 사용자 관리 |
| `community.docker` | `wordpress.yml`, `backup.yml`의 Docker 및 Compose 관리 |

## 6.2 설치

```bash
ansible-galaxy collection install -r requirements.yml
```

설치 후 Collection 목록을 확인하였다.

```bash
ansible-galaxy collection list | grep -E "ansible.mysql|community.docker"
```

Collection 설치 후 `database.yml` 문법 검사가 정상 통과하였다.

```text
playbook: playbooks/database.yml
```

기존 수동 구성과 Playbook 설정을 일치시키기 위해 `group_vars/all.yml`의 DB 변수를 다음과 같이 적용하였다.

이 설정은 기존 MariaDB 구성과 일치한다.

```text
MariaDB 수신 주소: 0.0.0.0:3306
```

기존 백업 및 관리용 DB 계정은 `database.yml`의 관리 대상이 아니므로 유지된다.

# 10. WordPress Compose 템플릿 오류

Web1에 `wordpress.yml`을 제한 실행하는 과정에서 Compose 설정 검증 오류가 발생하였다.

문제는 `wordpress-compose.yml.j2`에 다음 Ansible 변수 이름들이 Compose 최상위 키로 들어간 것이었다.

```text
wordpress_base_image
wordpress_image_name
wordpress_image_tag
wordpress_container_name
wordpress_host_port
wordpress_data_volume
```

Docker Compose 파일에서는 이러한 변수들을 최상위 설정으로 정의할 수 없다.

템플릿을 다음 구조로 수정하였다.

```yaml
---
services:
  wordpress:
    build:
      context: .
      args:
        WORDPRESS_BASE_IMAGE: "{{ wordpress_base_image }}"

    image: "{{ wordpress_image_name }}:{{ wordpress_image_tag }}"
    container_name: "{{ wordpress_container_name }}"
    restart: unless-stopped

    ports:
      - "{{ wordpress_host_port }}:80"

    environment:
      WORDPRESS_DB_HOST: "{{ hostvars[groups['db'][0]].ansible_host }}:{{ mariadb_port }}"
      WORDPRESS_DB_NAME: "{{ wordpress_db_name }}"
      WORDPRESS_DB_USER: "{{ wordpress_db_user }}"
      WORDPRESS_DB_PASSWORD: "{{ wordpress_db_password }}"

    volumes:
      - wordpress_data:/var/www/html

    healthcheck:
      test:
        - CMD-SHELL
        - curl -fsS http://127.0.0.1/ || exit 1
      interval: 30s
      timeout: 10s
      retries: 5
      start_period: 60s

volumes:
  wordpress_data:
    name: "{{ wordpress_data_volume }}"
```

수정 후 Compose 파일 검증 단계는 통과하였다.

# 12. 80번 포트 충돌 발생

새 `dandelion-wordpress` 컨테이너를 시작하는 단계에서 다음 오류가 발생하였다.

```text
failed to set up container networking
Bind for 0.0.0.0:80 failed: port is already allocated
```

실행 결과:

```text
web1-node: ok=5 changed=1 unreachable=0 failed=1
```

현재 확인된 상태는 다음과 같다.

```text
dandelion-wordpress:v1 이미지 빌드 완료
wordpress_default 네트워크 생성 완료
dandelion_wordpress_data 볼륨 생성 완료
dandelion-wordpress 컨테이너 생성됨
컨테이너 시작 실패
실패 원인: Web1의 호스트 80번 포트 사용 중
```

이 시점에서는 80번 포트를 점유한 기존 프로세스 또는 컨테이너를 아직 확인하거나 정리하지 않았다.
