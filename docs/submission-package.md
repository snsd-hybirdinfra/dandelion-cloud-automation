# Submission Package

## 1. 제출 목적

본 문서는 Team Dandelion 1차 기본 프로젝트의 최종 제출 산출물 목록을 정리한다.

GitHub Repository는 코드와 문서의 원본 관리 공간으로 사용하고, Google Drive는 최종 제출 패키지 보관 공간으로 사용한다.

## 2. Google Drive 제출 폴더 구조

~~~text
Dandelion_1차_기본프로젝트/
├── 01_결과보고서/
├── 02_시연영상/
├── 03_소스코드/
├── 04_작업일지_회의록/
├── 05_구현결과_캡처/
└── 06_기타자료/
~~~

## 3. 제출 산출물 체크리스트

| 구분 | 제출 자료 | 형식 | 상태 |
|---|---|---|---|
| 결과보고서 | 프로젝트 결과보고서 | PPTX | 준비 중 |
| 결과보고서 | 프로젝트 결과보고서 | PDF | 준비 중 |
| 시연 영상 | 기능별 시연 영상 | MP4 | 준비 중 |
| 소스 코드 | 최종 GitHub 소스 코드 압축본 | ZIP | 준비 중 |
| 소스 코드 | GitHub Repository URL | TXT | 준비 중 |
| 작업 일지 | 작업 일지 / 회의록 | MD 또는 DOCX | 준비 중 |
| 구현 캡처 | 인프라 / 서버 / Ansible / 검증 캡처 | PNG | 준비 중 |
| 기타 자료 | README, project-status, validation-summary | MD 또는 PDF | 준비 중 |

## 5. 제출 기준

최종 제출 시 Google Drive에는 결과보고서, 시연 영상, 소스 코드, 작업일지, 기타 자료를 모두 포함한다.

GitHub에는 프로젝트 원본 코드와 문서를 유지하고, Google Drive에는 평가자가 바로 확인할 수 있는 최종 제출본을 정리한다.
